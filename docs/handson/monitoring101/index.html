Monitoring 101 (Hands On Instructions)

Everyone should get login information for a Datadog account they can use for this session. If you haven’t received this information (and you are here in the room with us), contact the instructors. This will include a username and password for a Datadog account, as well as the IP Address for the load balancer of your cluster of servers.

  1. Login to your Demo Datadog Account. Note: Use this demo even if you have the credentials for your company’s account since it has been setup to collect data from the demo environment
  2. Navigate to the Metrics Explorer (https://app.datadoghq.com/metric/explorer).
  3. Type haproxy in the Graph textbox.
    This filters down the metrics available to just the ones with haproxy in the name.
  4. Select one of the metrics available, for instance haproxy.backend.response.2xx
  5. Notice the Over dropdown is a list of all the tags relevant to this metric.
  6. If you choose One graph per:host you will only see a single graph, since you have only one haproxy host.
  7. Change the metric to apache.net.request_per_s and set One graph per to host. You now have 3 graphs since your environment includes 3 webservers.
  8. Navigate to Dashboards and create a new Dashboard. Enter First Test for the name and choose New TimeBoard. We will talk about the differences soon.
  9. Notice the different visualization types. We have already talked about each of these.
  10. Drag and drop a Timeseries graph onto the canvas.
  11. Choose apache.net.request_per_s and avg by host.
  12. Try out the different Display types (Lines, Area, Bar) to see the differences.
  13. Add a Query Value viz to the canvas using the same metric.
  14. Add a Heat Map with the same metric.
  15. Repeat with the Distribution, Top List, Change, and Hostmap.
  16. Which graphs work well for this metric with 3 hosts? Take a look at the instructor’s dashboard to see how this is different for all of your hosts on one dashboard.
  17. Navigate to the Host Map. Click on one of the webservers running apache.
  18. Notice the tags assigned to the host.

Questions

  1. In this hands-on exercise, you created a heat map. Was it interesting? How could it be more interesting?
  2. What is the width in pixels for each graph? How many data points are shown in that space? Does this change for the different sizes on the graph? How about as you zoom in and out.
  3. How do you adjust how data is aggregated over time?
  4. How do you adjust how data is aggregated over space?
